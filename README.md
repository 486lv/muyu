# 木鱼：万法归咚 / Grand Muyu Simulator

这不再只是一个“按回车功德 +1”的电子木鱼，而是一个基于 C++17 的命令行修行模拟器。核心仍然是敲木鱼，但现在围绕它扩展出了门派、境界、寺院建设、任务、成就、随机事件、云游、打坐、诵经和长期存档。

## 核心玩法

你从一个刚入门的修行者开始，通过敲木鱼积累功德，建设寺院，吸引信众，获得灵玉，提升境界，最终抵达“万法归咚”。

游戏包含：

- **敲木鱼系统**：回车敲击，`n 50` 连续敲击，连击越高收益越高。
- **境界系统**：从“凡心未净”一路提升到“万法归咚”。
- **门派开局**：云隐寺、赛博禅院、香火道场，不同路线有不同初始加成。
- **寺院建设**：升级木鱼、香炉、蒲团、晨钟暮鼓、藏经阁。
- **资源系统**：当前功德、累计功德、信众、灵玉、专注。
- **行为系统**：敲木鱼、诵经、打坐、云游。
- **随机事件**：香客上山、灵玉凝结、藏经阁顿悟、杂念打断、钟鼓自鸣。
- **每日任务**：每天刷新功课，完成后可领取功德、信众和灵玉。
- **成就碑**：敲击、功德、连击、信众、灵玉、云游等长期目标。
- **存档系统**：自动保存为 `muyu_save.txt`，支持长期成长。

## 快速开始

### CMake 构建

```bash
cmake -S . -B build
cmake --build build
```

运行：

```bash
./build/muyu
```

Windows / Visual Studio 通常运行：

```powershell
.\build\Debug\muyu.exe
```

### g++ 直接编译

```bash
g++ -std=c++17 -Wall -Wextra -pedantic src/main.cpp -o muyu
./muyu
```

## 命令说明

### 核心命令

| 命令 | 作用 |
| --- | --- |
| 直接回车 | 敲一次木鱼 |
| `n <次数>` | 连续敲木鱼，例如 `n 50` |
| `chant` | 诵经，消耗专注，获得功德和信众 |
| `meditate` | 打坐，恢复专注并获得少量信众 |
| `pilgrimage` | 云游，有风险但收益较高 |

### 经营命令

| 命令 | 作用 |
| --- | --- |
| `status` | 查看寺院总览 |
| `shop` | 查看寺院建设商店 |
| `buy <编号/id>` | 购买升级，例如 `buy 1` 或 `buy fish` |
| `quests` | 查看今日任务 |
| `claim` | 领取已完成任务奖励 |
| `achievements` | 查看成就碑 |
| `rest` | 进入下一天，刷新任务节奏 |

### 系统命令

| 命令 | 作用 |
| --- | --- |
| `save` | 手动保存 |
| `reset` | 重置存档，需要输入 YES 确认 |
| `help` | 查看帮助 |
| `quit` / `exit` | 保存并退出 |

## 成长路线

境界大致如下：

1. 凡心未净
2. 初闻钟声
3. 寺门行者
4. 禅院执事
5. 功德长老
6. 云端罗汉
7. 赛博菩萨
8. 万法归咚

每次敲击、诵经、云游、完成任务都会推动你向更高境界前进。

## 设计方向

这个项目现在的目标不是“做一个按钮”，而是做一个可持续扩展的命令行修行世界。后续可以继续扩展：

- 多文件架构：拆出 `GameState`、`SaveSystem`、`QuestSystem`、`ShopSystem`。
- 图形界面：Qt / SFML / SDL 木鱼界面。
- 音效系统：敲击播放木鱼声。
- 剧情系统：章节、NPC、寺院事件链。
- 数据驱动：把商店、成就、任务放进 JSON。
- 排行榜：每日功德、总功德、最快成佛记录。
- 自动修行：挂机产出、建筑收益、离线收益。
